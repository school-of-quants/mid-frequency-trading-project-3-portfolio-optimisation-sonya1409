## ВНИМАНИЕ!

Внимательно изучите код бейзлайна, если хотите дорабатывать его, а не строить свою модель с нуля.
В особенности обратите внимание на комментарии по ходу кода: с некоторыми предположениями вы можете не согласиться.

Из-за того, что данных очень много, фаза получения данных занимает довольно долгое время.
Рекомендуется при отладке пайплайна запустить ее один раз, а тестирование модулей `train.py` и `run_backtest.py` проводить не из `main.py`.
Yfinance не дает скачать все требуемые котировки. Для недостающих бумаг найдите иной источник.

### Структура проекта

```bash
equity_project/
│
├── equity_project/             # Основной код проекта
│   ├── artifacts/
│   │   ├── metrics/
│   │   │   └── backtest_metrics.json  # Бэктестовые метрики стратегии, создается в src/run_backtest
│   │   └── plots/       
│   │       └── pnl.png         # График PnL стратегии, создается в src/run_backtest
│   ├── data/
│   │   ├── pony/               # Вспомогательные датасеты. В нашем случае это перечень бумаг, входивших в состав индекса S&P500 на протяжении последних  десятилетий
│   │   ├── raw/                # Исходные OHLC котировки бумаг, входящих в индекс S&P500 на протяжении рассматриваемого периода
│   │   └── processed/          # Обработанные датафреймы с фичами и таргетом для ML модели
│   └── src/                    # Исходный код
│       ├── get_data.py         # Здесь скачиваются данные
│       ├── run_backtest.py     # Здесь происходит инференс твоей стратегии
│       ├── train.py            # Здесь обучается ML модель, которая генерирует сигналы на покупку и продажу бумаг
│       └── utils.py            # Вспомогательные модули
├── main.py                     # Основной файл проекта, запускающий цикл стратегии от обучения до оценки метрик
├── config.yaml                 # Конфиг с требованиями к бэктесту
├── pyproject.toml              # Конфиг проекта, нужен нам для Poetry
│
└── README.md                   # Файл, который ты сейчас читаешь
```


## Установка и запуск

### Требования

Надеюсь, у тебя уже установлен Python. Если нет, то сделай это по [инструкции](https://practicum.yandex.ru/blog/kak-ustanovit-python-na-kompyuter/)

Для запуска проекта необходимо установить зависимости. Для этого будем использовать Poetry

```bash
pip install poetry==1.8.5
```

Следом ставим зависимости проекта
```bash
poetry install
```

Если стало интересно, то можно:
- [Почитать про Poetry](https://python-poetry.org/)
- [Посмотреть про Poetry и не только](https://ods.ai/tracks/mlops3-course-spring-2024/blocks/2e7ae437-1ada-45e1-a2c8-0f7e0a656411) 

## Что нужно сделать

Полные требования к проекту будут высланы вам в виде отдельного файла.

Вы можете модифицировать данный шаблон в случае необходимости, однако верхнеуровнево вам необходимо:

1. Определить ML модель для вашего проекта в `src/train.py`. 
2. Определить правила входа и выхода на основании скоров модели, а также предложить способ организации портфеля на их основе
3. Тщательно задокументировать ваш код
    - Прописать docstrings для каждой определенной функции и каждого введенного класса
    - Не стесняйтесь также писать однострочные или даже многострочные коммментарии по ходу кода
    - Из вашей документации должно быть понятно, что происходит в проекте
4. Закоммитить изменения в вашу локальную копию репозитория. Закомитьте также ваши данные, модель и артефакты (это плохая практика, но просим вас сделать так)

P.S. За использование LLM при написании проекта будем наказывать. Это образовательный проект, вы тут учитесь думать головой и писать разумный код. Не делегируйте процесс своего развития машине, пожалуйста.
